# Image 图片

增强型的图片组件，支持加载状态处理、失败后的后备显示（Fallback）以及延迟加载。基于 Kobalte 的 Image 核心构建，确保在各种网络状况下都有良好的用户体验。

## 引入

import { Image } from "solid-element-ui";

## 代码演示

### 1. 基础用法
展示一张普通的图片，并处理加载中的占位逻辑。

http://googleusercontent.com/image_collection/image_retrieval/11039202355893255513_0

<Image src="https://images.unsplash.com/photo-1511447333015-45b65e60f6d5" alt="星云图">
  <Image.Img />
  <Image.Fallback>正在加载...</Image.Fallback>
</Image>

### 2. 头像模式 (Fallback)
当图片加载失败或地址无效时，显示自定义的后备内容（如文字缩写或占位图标）。

<Image src="/invalid-path.jpg">
  <Image.Img class="h-12 w-12 rounded-full object-cover" />
  <Image.Fallback class="flex h-12 w-12 items-center justify-center rounded-full bg-zinc-200 text-sm">
    JD
  </Image.Fallback>
</Image>

### 3. 自定义加载延迟
你可以通过 delayMs 属性设置后备内容出现的延迟时间，避免在快速网络下出现闪烁。

<Image src="https://example.com/large-image.png" fallbackDelay={600}>
  <Image.Img />
  <Image.Fallback>
    <div class="h-40 w-full animate-pulse bg-zinc-100" />
  </Image.Fallback>
</Image>

## API 参考

### Image (Root)

| 属性 | 说明 | 类型 | 默认值 |
| :--- | :--- | :--- | :--- |
| src | 图片资源地址 | string | - |
| alt | 图片描述文本 | string | - |
| fallbackDelay | 后备内容显示的延迟时间 (ms) | number | 0 |
| onLoadingStatusChange | 加载状态改变时的回调 | (status: 'idle' \| 'loading' \| 'loaded' \| 'error') => void | - |

### Image.Img

| 属性 | 说明 | 类型 | 默认值 |
| :--- | :--- | :--- | :--- |
| class | 图片标签的样式类名 | string | - |
| loading | 原生延迟加载属性 | 'lazy' \| 'eager' | 'lazy' |

### Image.Fallback

| 属性 | 说明 | 类型 | 默认值 |
| :--- | :--- | :--- | :--- |
| class | 后备容器的样式类名 | string | - |

## 无障碍 (Accessibility)

- 替代文本: 通过 alt 属性确保屏幕阅读器能理解图片内容。如果是装饰性图片，建议将 alt 设为空字符串。
- 状态同步: 自动管理图片的可见性，确保只有在加载成功后才显示 Img，否则显示 Fallback。
- 角色定义: Fallback 容器在必要时可作为图片内容的文字替代方案。
